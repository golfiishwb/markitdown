# iOS/macOS WebView App (SwiftUI)

This example shows how to wrap the provided HTML into a SwiftUI app that runs on iOS and macOS (including Mac Catalyst). The app loads a local `index.html` from the app bundle in a `WKWebView`.

## What’s included

- `FlashcardsApp.swift`: App entry point.
- `ContentView.swift`: Hosts a single `WebView`.
- `WebView.swift`: `WKWebView` bridge for SwiftUI.
- `Resources/index.html`: Your provided HTML file.

## Quick start (Xcode)

1. Open Xcode → **File → New → Project…**.
2. Choose **App** (iOS) and check **Include macOS** (for a universal iOS+macOS app) or **Mac Catalyst**.
3. Name it `FlashcardsApp`.
4. Replace the generated Swift files with the ones in `FlashcardsApp/` from this folder.
5. Drag `Resources/index.html` into your project’s **Resources** (make sure **Copy items if needed** is checked and **Add to targets** is selected).
6. Build & run on iOS Simulator or macOS.

## Notes

- If you need offline use, keep the `index.html` self-contained. Remote resources (CDNs) require network access.
- For custom fonts or assets, include them in the app bundle and reference them with relative paths.
- You can enable App Transport Security exceptions if you load non-HTTPS resources (not recommended).

## Deploying to iOS

- Use the iOS target generated by Xcode.
- For App Store distribution, follow Apple’s signing and provisioning requirements.
